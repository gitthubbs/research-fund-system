**高校科研经费管理与决策支持系统：后端开发执行手册**

### **0\. 核心约束（红线）**

* **禁止变更表结构**：严格使用已定的 6 张表。  
* **技术栈锁定**：Spring Boot \+ MyBatis-Plus，不使用 XML Mapper。  
* **业务聚焦**：仅实现“预算、支出、统计”闭环，**禁止**实现审批、报销、流转逻辑。  
* **接口规范**：纯 JSON 返回，适配 ECharts 数据结构。

### **1\. 开发环境**

* **JDK**: 17 / 11  
* **DB**: MySQL 8.x  
* **ORM**: MyBatis-Plus (BaseMapper \+ @Select)  
* **JSON**: Jackson (Spring Boot 默认)

### **2\. 统计维度与逻辑算法（核心开发点）**

| 统计指标 | 计算公式 / 逻辑 | 建议实现方式 |
| :---- | :---- | :---- |
| **执行率 (Execution Rate)** | $\\sum \\text{expenditure.amount} / \\sum \\text{budget.amount}$ | Service 层聚合计算 |
| **支出分布 (Exp Dist)** | GROUP BY category\_id 汇总 amount | Mapper 层 @Select \+ SUM |
| **预算结构 (Bud Dist)** | GROUP BY category\_id 汇总 budget\_amount | Mapper 层 @Select |
| **月度趋势 (Trend)** | GROUP BY DATE\_FORMAT(exp\_date, '%Y-%m') | Mapper 层 SQL 格式化 |

### **3\. Mapper 层扩展定义 (No XML)**

Agent 需在各 Mapper 接口中通过注解实现以下逻辑：

Java

// 示例：FundExpenditureMapper.java  
@Select("SELECT category\_id, SUM(amount) as total FROM fund\_expenditure WHERE project\_id \= \#{projectId} GROUP BY category\_id")  
List\<Map\<String, Object\>\> selectExpenditureByCategory(Long projectId);

@Select("SELECT DATE\_FORMAT(expenditure\_date, '%Y-%m') as month, SUM(amount) as amount " \+  
        "FROM fund\_expenditure WHERE project\_id \= \#{projectId} GROUP BY month ORDER BY month ASC")  
List\<Map\<String, Object\>\> selectMonthlyTrend(Long projectId);

### **4\. 接口契约 (API Blueprint)**

* **项目概览**: GET /api/statistics/execution-rate/{projectId}  
  * 返回：{ "totalBudget": 0.0, "totalSpent": 0.0, "rate": 0.0 }  
* **支出分类**: GET /api/statistics/expenditure/category/{projectId}  
  * 返回：List\<CategoryAmountVO\>  
* **趋势分析**: GET /api/statistics/expenditure/trend/{projectId}  
  * 返回：List\<TrendVO\>

### **5\. 执行指令 (Direct Instructions to Agent)**

1. **第一阶段 (Persistence)**: 确认 Entity 字段与 MySQL 类型（尤其是 BigDecimal）对齐。  
2. **第二阶段 (Logic)**: 在 FundStatisticsServiceImpl 中完成多表数据的读取与百分比计算。处理 NULL 值，确保项目无支出时返回 0 而非异常。  
3. **第三阶段 (Interface)**: 编写 Controller，确保返回格式为 Result\<T\> 统一包装类。  
4. **第四阶段 (Snapshot)**: 实现手动触发接口，将当前统计结果持久化至 fund\_statistics\_snapshot 表。
