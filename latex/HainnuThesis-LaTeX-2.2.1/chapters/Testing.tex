\chapter{系统测试与分析}

这个章节主要是记录系统做的一些测试工作。测试采用了黑盒测试的办法，模拟了科研人员和管理人员在系统里的操作，旨在验证这些功能是否达到了之前设计的要求。

\section{测试环境与测试策略}
测试过程中，环境尽量保持了与开发状态的一致性。后端仍旧运行在容器中，数据库选用 MySQL 8.0。笔者重点针对报销流程和预警逻辑进行了多轮测试，因为这两个环节的准确性直接关系到财务管理的严肃性。

\section{测试后的分析}
从这些用例的执行情况来看，系统基本上达到了预期的设计目标。

首先是逻辑的严密性。程序内置的“财务围栏”能有效阻断超预算的报销申请，这对规范经费使用非常有保障。另外就是预警功能，经过反复测试，逻辑引擎对执行率不正常的项目确实表现出了较高的敏感度。

至于安全性，基于 Spring Security 的底层防护跑得挺稳。测试中尝试利用科研人员账号越权调用管理接口，均被系统准确拦截。总的来看，系统已经具备了部署上线的基本条件。

\section{本章小结}
本章对主要的测试数据进行了归纳分析。测试结果表明，各功能模块运行正常，特别是在多维预警逻辑和财务拦截机制上，系统表现出了较强的健壮性。这充分验证了前述章节设计方案在技术实践层面的可行性。
