# 二、开发原则（必须严格遵守）

1. **禁止破坏已有代码**

    * 不得删除现有字段、接口、类
    * 不得重命名已有接口
    * 只能“新增或扩展”

2. **所有新增或修改代码必须标注**

   ```java
   // ★ 新增
   // ★ 修改
   ```

3. **保持现有项目结构**

    * 后端遵循 controller / service / mapper 分层
    * 前端遵循 views / api / components 结构

4. **优先复用已有代码**

    * 如果已有类似接口 → 扩展
    * 不重复造轮子
